--- layout: post title: gtk 3 scrollbar in debian (thunderbird, icedove) date: '2017-07-16T01:16:00.004-07:00' author: Alex Madon tags: unix modified_time: '2017-07-21T01:03:35.152-07:00' blogger_id: tag:blogger.com,1999:blog-7521237649117347170.post-5362716547549608009 blogger_orig_url: http://tuxunix.blogspot.com/2017/07/gtk-3-scrollbar-in-debain-thunderbird.html --- To get the scrollbar arrows back:

.config/gtk-3.0/gtk.css

* {
-GtkRange-slider-width: 18px;
-GtkScrollbar-has-forward-stepper: 1;
-GtkScrollbar-has-backward-stepper: 1;
-GtkScrollbar-has-secondary-forward-stepper: 1;
-GtkScrollbar-has-secondary-backward-stepper: 1;
-GtkScrollbar-min-slider-length: 30px;
}
.scrollbar.slider.vertical,
.scrollbar.button.vertical {
border-radius: 4px;
}



also if you want to page down/up when right clicking in the scrollbar, have:

~/.config/gtk-3.0# cat settings.ini
[Settings]
gtk-primary-button-warps-slider = false



see also

https://wiki.gnome.org/Projects/GTK+/Inspector
The debugger is disabled by default. To enable it run in a terminal:

gsettings set org.gtk.Settings.Debug enable-inspector-keybinding true

And launch it pressing on the keyboard Control-Shift-I or Control-Shift-D.

If you don't want to use the shortcuts, you can also run it temporarily directly when running your app with:

GTK_DEBUG=interactive your-app
